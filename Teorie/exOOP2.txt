25. Activitate la tema MVC
1. Cand este nevoie de a aplica modelul MVC?
Modelul MVC este nevoie de a fi aplicat pentru a desparti logica site-ului de partea de design, previne refolosirea codurilor si organizarea acestora DRY (Don't Repeat Yourself). Datorita separarii vom avea o structura mai solida.

2. Care este esenta modelului MVC?
MVC - Model View Controller
Model - manipuleaza operatiunile logice si de utilizare de informatie (trimisa dinainte de catre rangul sau superior) pentru a rezulta de o forma u?or de inteles.
View - reprezentarea grafica, sau mai bine zis, exprimarea ultimei forme a datelor: interfata grafica ce interactioneaza cu utilizatorul final. Rolul sau este de a evidentia informatia obtinuta pana ce ea ajunge la controlor.
Controller - putem controla accesul la aplicatia noastra. Pot fi fisiere, scripturi (eng. scripts) sau programe, in general orice tip de informatie permisa de interfata. In acest fel putem diversifica continutul nostru de o forma dinamica si statica, in acelasi timp.

3. Argumentati de ce MVC este un pattern de programare.
MVC este un model de programare deoarece poate separa partile logice ale unei aplicatii si le poate crea separat una de cealalta. Adica, daca scrieri blocuri de cod independente, le puteti modifica dupa cum este necesar fara a afecta alte blocuri de cod.
